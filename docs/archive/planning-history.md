# PFE 规划演进（归档）

更新时间：2026-04-21

本文档用于保留项目规划层面的历史上下文。

早期仓库里曾分别保存：

- 最初的阶段路线图
- 2026-04-12 的 v2 开发规划

这些文档包含大量阶段性推演、周计划和当时仍未落地的假设。对公开仓库来说，逐份保留会让文档层显得过重，也容易让外部读者把历史计划误读成当前状态。

因此这里把它们压缩为一份历史摘要。

## 1. 最初规划的主线

最早的路线图把项目分成 4 个阶段：

1. `Phase 0`
   - 证明本地 LoRA 微调能产生可感知的风格变化
   - 跑通 `generate -> train -> eval`
2. `Phase 1`
   - 从脚本集合走向可用工具
   - 补齐信号采集、样本整理、DPO、serve、CLI 和端到端测试
3. `Phase 2`
   - 深化个性化质量
   - 引入用户画像、冲突检测、Replay、Teacher 融合、安全与可观测性
4. `Phase 3`
   - 走向生态开放
   - 插件体系、多模型支持、示例应用和文档站

这个规划的价值在于定义了 PFE 的长期产品形状：

- 不是单一聊天应用
- 而是一个本地持续个性化微调引擎

## 2. 2026-04-12 的 v2 规划

到 2026-04-12，项目已经不再是骨架状态，因此规划发生了明显调整：

- 引入 `Phase 0.5` 这个中间阶段，用来描述“真实训练落地”
- 明确 `train -> eval -> promote -> serve` 已在真实路径上打通
- 把后续重点从“证明 loop 能跑”转移到“自动闭环、控制面和交互式工作台”

v2 规划的重要修正有两点：

1. 对项目实际完成度的判断更贴近代码现状
2. Phase 1 / Phase 2 的目标开始从“搭能力”转向“把能力做成可用闭环”
