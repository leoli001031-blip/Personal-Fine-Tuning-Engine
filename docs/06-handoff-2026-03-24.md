### 2.2 核心约束已落地

以下文档约束已在代码中落地，不应被后续开发破坏：

- 默认模式是 `strict_local`
- OpenAI 兼容仅指推理接口，不等于个性化闭环
- 个性化闭环依赖 Signal SDK 或 `/pfe/signal`
- 数据模型必须保留：
  - `request_id`
  - `session_id`
  - `adapter_version`
  - `source_event_ids`
- 自动 DPO 配对必须依赖完整事件链，不能猜配对
- adapter lifecycle 必须有：
  - `training`
  - `pending_eval`
  - `promoted`
  - `failed_eval`
  - `archived`
- `latest` 只能通过 `promote()` 更新
- adapter 产物必须使用标准目录和 `adapter_manifest.json`
- `llama.cpp` 路径必须走 `gguf_merged` 导出，不能直接加载 LoRA safetensors
- 蒸馏样本保留 provenance：
  - `teacher_model`
  - `teacher_prompt_version`
  - `generation_config`
  - `dataset_split`
- Judge 评测优先使用 holdout / test split
- `test` split 不参与训练，也不得进入 Replay Buffer
