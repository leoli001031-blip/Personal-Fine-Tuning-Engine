|------|----------|------|

### 2.2 信号采集 (ChatCollector)

| 功能 | 状态 | 说明 |
|------|------|------|
| 采纳信号 | ✅ | 用户未编辑继续对话 |
| 拒绝信号 | ✅ | 删除/清空回复 |
| 编辑信号 | ✅ | 计算编辑距离，分级置信度 |
| 重生成信号 | ✅ | 请求重新生成 |
| 显式反馈归一化 | ✅ | 无 pending interaction 时仍统一到标准信号类型 |
| 路径隔离 | ✅ | 审计日志和相关持久化默认落在 `PFE_HOME` 下 |

**文件**: `pfe_core/collector/chat_collector.py`

**信号置信度规则**:
| 信号类型 | 置信度 |
|---------|--------|
| 强采纳 (响应<5s) | 0.9 |
| 采纳 | 0.7 |
| 弱采纳 (响应>60s) | 0.4 |
| 编辑距离<20% | 0.6 |
| 编辑距离20-50% | 0.8 |
| 编辑距离>50% | 0.9 |
| 拒绝/删除 | 0.95 |
| 重生成 | 0.85 |

### 2.3 自动闭环策略

**文件**: `pfe_core/trainer/policy.py`

| 策略 | 默认值 | 说明 |
|------|--------|------|
| Train Trigger | 50 samples | 信号数量阈值 |
| Cooldown | 60 min | 两次训练间隔 |
| Eval Gate | 手动 | 自动触发评测 |
| Promote Gate | overall≥0.7 | 质量门槛 |
| Compare Gate | 改进≥0.05 | 新旧版本对比 |

### 2.4 可靠性机制

**文件**: `pfe_core/reliability.py` (902 行)

| 机制 | 说明 |
|------|------|
| Heartbeat | runner 定期心跳，检测僵死 |
| Lease | 任务防竞争，超时释放 |
| Restart 退避 | 指数退避，最大 300s |
| Recovery | checkpoint 断点恢复 |
| Alert | 三级阈值告警 |

### 2.5 交互式工作台 (pfe console)

```bash
pfe console
```

**支持的命令**:
- `/do trigger-train` - 触发训练
- `/approve <id> [note]` - 批准队列项
- `/reject <id> [note]` - 拒绝队列项
- `/promote [note]` - 提升候选版本
- `/rollback [version]` - 回滚版本

## 5. 快速开始

### 5.1 环境初始化

```bash
cd ~/Desktop/PFE

# 创建虚拟环境
python3.11 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 安装依赖
pip install -e .
pip install torch transformers peft accelerate trl datasets

# 验证
pfe doctor
```

### 5.2 完整流程测试

```bash
# 1. 初始化
pfe init --base-model Qwen/Qwen2.5-3B-Instruct

# 2. 生成冷启动数据（bootstrap）
pfe generate --scenario life-coach --num 50

# 2b. 可选 Teacher / Distill 入口
# 注意：若未配置真实 teacher path，当前工作流仍可能回退到模板/合成样本
pfe distill --teacher-model gpt-4o --scenario life-coach --num 50

# 3. 训练 (SFT)
pfe train --method qlora --epochs 3

# 4. 评测
pfe eval --adapter latest --num-samples 20

# 5. 提升版本
pfe adapter promote <version>

# 6. 启动服务
pfe serve --port 8921
```

### 5.3 使用 ChatCollector

```bash
# 启动服务和 daemon
pfe serve &
pfe daemon start

# 打开交互式控制台
pfe console

# 进行对话，系统会自动采集信号
# 信号达到阈值后自动触发训练

# 查看采集统计
pfe collect status
```

## 6. 核心约束（不可违反）

以下约束已在代码中落地，后续开发必须遵守：

1. **`strict_local` 为默认模式** - 云端功能需显式授权
2. **`latest` 只能通过 `promote()` 更新** - 禁止直接修改
3. **`test` split 不参与训练** - 也不得进入 Replay Buffer
4. **DPO 配对必须基于完整事件链** - 不能猜测配对
5. **llama.cpp 必须走 GGUF 导出** - 当前已验证 `base GGUF + LoRA GGUF + --lora`
6. **OpenAI 兼容接口 ≠ 个性化闭环** - 推理接口只是出口
7. **蒸馏样本保留完整 provenance** - teacher model / prompt version / generation config
8. **adapter 产物使用标准目录** - 必须包含 `adapter_manifest.json`

## 8. 迁移到新电脑

### 8.1 复制内容

```bash
# 原电脑：清理后打包
cd ~/Desktop/PFE
rm -rf .venv __pycache__ .uv-cache .pfe/logs
find . -name "*.pyc" -delete
tar czvf pfe-backup.tar.gz --exclude='.venv' --exclude='__pycache__' .

# 复制到新电脑
scp pfe-backup.tar.gz new-mac:~/Desktop/
```

### 8.2 新电脑初始化

```bash
cd ~/Desktop
tar xzvf pfe-backup.tar.gz
mv PFE PFE

cd PFE
python3.11 -m venv .venv
source .venv/bin/activate
pip install -e .
pip install torch transformers peft accelerate trl datasets

# 验证
pfe doctor
```

### 8.3 可选：重新下载模型

如果模型文件太大不想复制：

```bash
# 新电脑上重新下载
pfe init --base-model Qwen/Qwen2.5-3B-Instruct
```
